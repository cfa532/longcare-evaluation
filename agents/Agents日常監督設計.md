# Agents 日常監督設計

目的：讓日常營運紀錄自然累積成評鑑佐證，降低評鑑前補資料的壓力，並及早發現缺口。

## 一、Agent 角色分工

| Agent | 核心任務 | 對應評鑑重點 | 產出 |
|---|---|---|---|
| 日常紀錄監督 Agent | 每日檢查排班、派案、異動、異常事件是否有紀錄。 | A3、A9、B項服務品質、D項權益保障 | 每日檢核紀錄、缺漏提醒 |
| 佐證資料管理 Agent | 將會議、訓練、健檢、保險、人力等資料分類到指標。 | A1-A9 全項 | 佐證文件索引、缺件清單 |
| 缺失改善追蹤 Agent | 追蹤查核缺失、前次評鑑建議、內部稽核缺口。 | A4、B8、D3、D4 | 改善追蹤表、逾期提醒 |
| 評鑑準備總管 Agent | 每週彙整風險、進度、下一步，維護專案狀態。 | 全部指標 | 週報、評鑑準備總表 |

## 二、日常監督原則

1. Agent 不取代主管或業務負責人，而是協助檢查紀錄是否完整。
2. 每次檢查都要留下紀錄，紀錄本身也是管理佐證。
3. 發現缺口時，不只提醒，還要寫入「問題追蹤表」。
4. 每個缺口需有負責人、期限、追蹤結果。
5. 所有資料依評鑑指標分類，避免評鑑前重新整理。

## 三、每日檢查範圍

每日下班前檢查：

| 檢查項目 | 對應指標 | 判斷標準 |
|---|---|---|
| 當日排班是否完整 | A9 | 有服務對象、照服員、時段、核准人。 |
| 調班、代班、更換服務人員是否留紀錄 | A9 | 有原因、通知對象、核准人、佐證位置。 |
| 新進人員是否有待補資料 | A5、A6、A7、A8 | 聘任、勞健保、登錄報備、健檢、職前訓練未漏。 |
| 是否發生異常或緊急事件 | A2、B8、D3 | 有事件紀錄、通報、原因分析、追蹤。 |
| 是否新增主管機關通知或查核事項 | A4 | 有建檔、負責人、期限。 |
| 是否有可歸檔佐證資料 | A1-A9 | 已放入索引或待歸檔清單。 |

## 四、每週檢查範圍

每週固定檢查：

| 檢查項目 | 對應指標 | 判斷標準 |
|---|---|---|
| 照服員每週工時是否合理 | A9 | 工時統計完整，異常有說明。 |
| 督導紀錄是否依期程進行 | A3 | 個別督導、團體督導進度可追蹤。 |
| 缺失改善是否逾期 | A4 | 逾期項目列入追蹤。 |
| 新進人員職前訓練是否在1個月內完成 | A8 | 未完成者列入補訓。 |
| 健檢、疫苗、保險名冊是否有異動 | A5、A7 | 新進或離職人員已更新。 |

## 五、每月檢查範圍

每月固定檢查：

| 檢查項目 | 對應指標 | 判斷標準 |
|---|---|---|
| 人員名冊與督導配置比例 | A6 | 每60位個案含未滿60位配置1位專任督導。 |
| 勞健保、勞退、保險佐證 | A5 | 每位在職員工有資料。 |
| A項佐證索引是否更新 | A1-A9 | 新增資料已歸檔。 |
| 服務品質或工作改善議題 | A3 | 可帶入下次行政聯繫會議。 |

## 六、每季檢查範圍

每季固定檢查：

| 檢查項目 | 對應指標 | 判斷標準 |
|---|---|---|
| 行政聯繫會議是否召開 | A3 | 至少每3個月一次，有紀錄與出席表。 |
| 年度業務計畫目標追蹤 | A1 | 有達成率、原因分析、改善策略。 |
| 工作手冊是否有需修訂事項 | A2 | 法規、流程、通報窗口異動已處理。 |
| 缺失改善完成率 | A4 | 追蹤改善率，目標100%。 |

## 七、Agent 回報格式

每次監督回報採下列格式：

1. 本次檢查日期。
2. 檢查範圍。
3. 已完成事項。
4. 發現缺口。
5. 需補件或改善事項。
6. 建議負責人。
7. 建議期限。
8. 對應評鑑指標。
9. 下一次追蹤日期。

## 八、使用限制

Agent 只能檢查專案資料夾內已提供或已建檔的資料。若實際營運資料尚未輸入，Agent 會提醒缺資料，但不能判斷現場實際是否已完成。

因此建議每日將新產生的表單、會議紀錄、排班異動、員工異動與主管機關通知放入本專案，或在對話中交給 Codex 代為整理。
